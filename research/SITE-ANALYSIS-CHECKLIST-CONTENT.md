### BEFORE YOU VISIT

- [ ] Google Street View - check historical imagery (toggle years). Note what changed.
- [ ] Aerial/satellite view - screenshot for base layer diagram
- [ ] Zoning map - check permitted use, height limits, setbacks, FAR
- [ ] Surrounding context - what's within 500m? Schools, transit, parks, commercial?
- [ ] Historical research - has the site been anything else before? Cultural significance?
- [ ] Precedent projects - find 2-3 buildings nearby or on similar sites

### ON SITE (bring phone + notebook)

- [ ] First impressions - write 3 words that describe how the site FEELS
- [ ] Entry points - where did you naturally enter? Where do others enter?
- [ ] Movement patterns - where do people walk? Where do they stop? Where do they avoid?
- [ ] Sun exposure - which areas are in sun vs shade RIGHT NOW? (note the time)
- [ ] Wind - can you feel it? From which direction? Blocked by anything?
- [ ] Sound - what can you hear? Traffic? Birds? People? Silence? Rate 1-5 noise level
- [ ] Views - best view FROM the site? Best view OF the site? Any views to protect/frame?
- [ ] Edges + boundaries - what defines the site edges? Walls, roads, fences, level changes?
- [ ] Vegetation - existing trees (mark locations + approx. height), ground cover
- [ ] Levels + topography - is it flat? Sloped? Any retaining walls or drainage?
- [ ] Photos - take from each corner, from center looking out, and standing at each edge looking in

### THE ANALYSIS (this is where most students stop too early)

**For EVERY piece of data you collected, complete this sentence:**

*"[DATA] means that [DESIGN IMPLICATION]"*

Examples:
- "Prevailing SW wind" → "Living spaces should face NE, OR use wind for natural ventilation"
- "Adjacent 4-storey building to the south" → "Ground floor in shadow after 2pm - use upper floors for daylight-dependent program"
- "Bus stop 50m east" → "Main entrance should face east to capture pedestrian flow from transit"
- "Heavy traffic noise from north road" → "Buffer zone/service spaces on north edge, living spaces set back or south-facing"

- [ ] Written design implications for climate data (sun, wind, rain)
- [ ] Written design implications for access + movement
- [ ] Written design implications for surrounding context
- [ ] Written design implications for views + sight lines
- [ ] Written design implications for noise + sound
- [ ] Written design implications for existing vegetation/features

### THE SYNTHESIS

Write your 3 site-driven design principles:

1. _______________________________________________
2. _______________________________________________
3. _______________________________________________

These 3 principles should guide EVERY major design decision in your project.

### DIAGRAM FORMAT

Your site analysis page should read: **LEFT = raw data/diagram → RIGHT = what it means for your design**

NOT just: sun path + wind rose + site photos + done.
